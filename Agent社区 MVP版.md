# AI Agent 社区小程序 MVP 版本 BRD+PRD+FSD 文档

## 第一部分：BRD 商业需求文档（MVP 聚焦）

### 1. 项目概述
*   **核心定位**：AI Agent 垂直领域的“避坑指南”与“资源中转站”。
*   **MVP 目标**：在 2 周内上线微信小程序，实现“教程查阅 + 资源下载 + 基础交流”闭环，验证用户对轻量化社区的需求。

### 2. 用户分析（MVP 聚焦）
*   **核心用户**：零基础/入门级开发者（占比 80%）。他们最痛苦的是环境配置报错和找不到适配的配置文件。
*   **次要用户**：进阶玩家。他们需要分享作品获得认同感。

### 3. 商业模式（MVP 阶段）
*   **完全免费**：MVP 阶段不设置任何付费墙，所有教程和资源免费开放，以快速积累种子用户。
*   **流量变现**：仅接入微信流量主（Banner/视频广告），作为后期维护服务器的成本补充。

## 第二部分：PRD 产品需求文档（MVP 功能裁剪）

### 1. 产品架构（四大模块简化为三大模块）
为了降低开发成本，MVP 版本将底部 Tab 简化为：**首页（聚合）、教程（内容）、我的（个人）**。
*   *注：资源广场并入首页推荐和教程附件；社区圈子暂时简化为教程下方的评论区。*

### 2. 核心功能需求清单

#### 2.1 首页模块（聚合页）
*   **轮播图 (Banner)**：展示最新重磅教程或公告（3-5 张）。
*   **金刚区 (快捷入口)**：新手入门、环境配置、报错排查、官方工具。
*   **热门榜单**：展示浏览量最高的 Top 5 教程和资源。
*   **全局搜索**：支持搜索教程标题和资源标签。

#### 2.2 教程中心模块（核心内容）
*   **分类筛选**：仅保留一级分类（如：Hermes, OpenClaw, Ollama, 通用环境）。
*   **教程详情**：
  *   富文本展示（图文混排）。
  *   **代码块一键复制**（核心痛点功能）。
  *   关联资源下载链接（直接在教程页下载相关配置/脚本）。
*   **互动反馈**：教程底部增加“点赞”和“评论/提问”区（替代独立的社区发帖功能）。

#### 2.3 个人中心模块
*   **微信一键登录**：获取头像、昵称。
*   **我的收藏**：收藏的教程列表。
*   **浏览历史**：最近看过的教程。
*   **意见反馈**：简单的表单提交，用于收集用户建议。

### 3. 砍掉的功能（MVP 不做）
*   ❌ 独立的社区发帖/私信功能（开发成本高，初期易产生垃圾信息）。
*   ❌ 复杂的积分/付费体系。
*   ❌ 资源上传功能（初期由管理员后台统一上传，保证质量）。
*   ❌ 消息通知系统。

## 第三部分：FSD 功能详细设计文档（开发落地）

### 1. 技术选型（保持先进性与轻量化）
*   **框架**：UniApp (Vue3 + Vite + TypeScript)。
*   **状态管理**：Pinia（管理用户信息、收藏状态）。
*   **UI 库**：Uni-ui 或 Wot-design-uni（轻量、美观、兼容性好）。
*   **数据请求**：Alova.js 或 封装 uni.request（带拦截器）。
*   **后端**：建议使用云开发（微信云托管/云函数）或轻量级 Node.js/Python 服务，减少运维成本。

### 2. 模块详细设计

#### 2.1 首页 (pages/index)
*   **布局**：顶部自定义导航栏 + 搜索框 -> 轮播图 -> 金刚区图标 -> 热门教程列表。
*   **逻辑**：
  *   `onLoad` 时并行请求 Banner 数据和热门教程数据。
  *   使用 `scroll-view` 实现下拉刷新。

#### 2.2 教程列表与详情 (pages/tutorial)
*   **列表页**：左侧/顶部 Tab 切换分类，右侧瀑布流或列表展示教程卡片。
*   **详情页**：
  *   使用 `rich-text` 或专门的 Markdown 渲染组件（如 `mp-html`）。
  *   代码块区域增加“复制”按钮，调用 `uni.setClipboardData`。
  *   底部固定“收藏”和“评论”按钮。

#### 2.3 个人中心 (pages/me)
*   **登录逻辑**：点击头像触发 `uni.login` 和 `getUserProfile`，获取 `code` 换取 `openid` 和 `session_key`。
*   **数据存储**：
  *   本地：`uni.storage` 存储 Token、用户基本信息。
  *   云端：关联用户 ID 存储收藏关系。

### 3. 数据库设计（极简版）

| 表名 | 字段说明 | 备注 |
| :--- | :--- | :--- |
| `users` | id, openid, avatar, nickname, create_time | 用户表 |
| `tutorials` | id, title, cover, category_id, content(markdown), view_count, like_count | 教程表 |
| `categories` | id, name, sort | 分类表 |
| `resources` | id, tutorial_id, name, url, download_count | 资源表（关联教程） |
| `comments` | id, tutorial_id, user_id, content, create_time | 评论表 |
| `favorites` | id, user_id, tutorial_id, create_time | 收藏表 |

### 4. 项目排期（MVP 10 天冲刺）

*   **Day 1-2**: 环境搭建，完成首页 UI、自定义导航栏、网络请求封装。
*   **Day 3-5**: 教程列表、详情页开发，重点攻克 Markdown 渲染和代码复制功能。
*   **Day 6-7**: 微信登录授权、个人中心、收藏功能开发。
*   **Day 8**: 评论功能开发，后台数据录入（准备 10-20 篇高质量种子教程）。
*   **Day 9**: 整体联调，Bug 修复，性能优化（图片压缩、分包）。
*   **Day 10**: 提交微信审核，准备发布文案。

### 5. MVP 成功标准
1.  **功能可用**：用户能顺畅看完一篇教程并复制代码。
2.  **性能达标**：首屏加载时间 < 1.5s。
3.  **内容填充**：上线时至少有 20 篇原创/整理的高质量教程。

---

### 给开发者的建议：
1.  **内容为王**：MVP 阶段，代码写得再花哨，如果没有好用的教程，用户也不会留存。前期请花 50% 的精力整理内容。
2.  **快速迭代**：不要追求完美，先上线。社区发帖、积分系统等复杂功能，等用户量达到 1000 人后再考虑加入。
3.  **注意合规**：小程序对“UGC（用户生成内容）”审核非常严格。MVP 阶段尽量弱化用户发帖，或者采用“先审后发”机制，避免因为内容违规导致下架。
